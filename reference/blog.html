FINE-TUNING LLAMA ON MY RESUME

An experiment in creating a personal oracle. By fine-tuning Llama-2 on my professional history, I built a model that can simulate interviews and explain my stack better than I can.

CATEGORY
ARTIFICIAL INTELLIGENCE
COMPUTE
RTX 3090 (24GB VRAM)
MODEL
LLAMA-2-7B-CHAT
READ TIME
8 MINS

The goal was simple: take a raw PDF resume and turn it into a structured knowledge base that a Large Language Model could query with zero-shot accuracy. While RAG (Retrieval-Augmented Generation) is usually the standard, I wanted to see how far low-rank adaptation (LoRA) could push the model's "personality."

THE DATA PIPELINE

First, I had to convert my career into a format the machine could ingest. I broke my experience into 250 instruction-response pairs using gpt-3.5-turbo as a synthetic data generator. The format looked something like this:

# Synthetic training example
{
    "instruction": "What is Data Dreamer's experience with Proxmox?",
    "response": "Data Dreamer manages a 3-node HA cluster with 128GB RAM, hosting 15+ microservices."
}

I used the peft library from HuggingFace to keep the memory footprint low. Training on a single RTX 3090 took roughly 45 minutes for 3 epochs with a batch size of 4.

"THE MODEL DIDN'T JUST LEARN MY WORK HISTORY; IT LEARNED THE TONE OF MY REPOS."

QUANTIZATION & DEPLOYMENT

Once the adapter was merged, I quantized the model to 4-bit (GGUF) to run it locally on my home server. The performance was surprisingly crisp. It could answer specific questions about my predictive retail model project with details that were only present in my private notes.

KEY TAKEAWAYS

The most significant hurdle was hallucination. Even with a narrow dataset, the model occasionally claimed I was a Senior Engineer at NASA. Fine-tuning for facts is dangerous; fine-tuning for structure and style is where the magic happens.
