The objective was simple: build a chatbot that doesn't just hallucinate my skills, but understands the nuances of my homelab architecture and data pipelines. By utilizing Low-Rank Adaptation (LoRA), I managed to train a specialized model on a single RTX 3090 in under 4 hours.

System Tip

Ensure your dataset is formatted in JSONL with strict 'instruction' and 'output' pairs for the most consistent behavioral alignment.

02 // DATASET PREPARATION

Garbage in, garbage out. I scraped my own GitHub commit messages, LinkedIn profile, and Notion project boards to create 500 high-quality instruction-response pairs.

DATA SOURCES USED

  • GitHub commit history — 3,200 messages across 14 repos
  • LinkedIn profile exports — experience, endorsements, summaries
  • Notion project docs — architecture notes and retrospectives
  • Personal blog posts — technical breakdowns from this very log

PROCESSING PIPELINE

  1. Normalize raw text — strip markdown, clean encoding artifacts
  2. Segment into instruction/output pairs using GPT-4 as annotator
  3. Deduplicate with MinHash LSH at 0.85 Jaccard threshold
  4. Final quality check — manual review of 10% random sample
  5. Export as JSONL, split 90/5/5 train/val/test
SAMPLE JSONL RECORD FORMAT
Each record follows the Alpaca instruction format: {"instruction": "...", "input": "", "output": "..."}. The instruction describes the query type (career, technical, personal) and the output is a first-person response grounded in actual project history. Avoid using generic phrasing—the model learns your voice through repetition of authentic output examples.

03 // LoRA CONFIGURATION

Using PEFT (Parameter-Efficient Fine-Tuning), we only update a small fraction of the model's weights. Here is the configuration block used for the training session: 

from peft import LoraConfig, get_peft_model

config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

# Initialize trainer with 4-bit quantization
model = get_peft_model(base_model, config)
"THE GOAL ISN'T JUST TO STORE DATA, BUT TO ENCODE INTUITION."
Hardware Alert

Training without 4-bit quantization (bitsandbytes) will likely result in OOM (Out of Memory) errors on consumer-grade hardware for 7B+ parameter models.

04 // INFERENCE TESTING

After the weights were merged, the results were startlingly accurate. When asked "How does the Data Dreamer handle ZFS pool failures?", the model correctly referenced my specific Proxmox setup rather than general documentation.

Note

Inference latency on RTX 3090 averaged 340ms per token at 4-bit quantization. For production deployment, consider serving via vLLM for 3–5× throughput improvement.

BENCHMARK RESULTS

  • Career factual accuracy — 94.2% vs base Llama at 31%
  • Homelab topology recall — 88.7% correct architecture references
  • Tone consistency — matches writing style in 91% of outputs
  • Hallucination rate — reduced to 4.1% from 47% baseline
RAW INFERENCE LOG — SAMPLE QUERY/RESPONSE
Q: "How do you handle storage failures in your homelab?"

A: "My storage stack runs ZFS on Proxmox with a 6-disk RAIDZ2 pool. On failure, I get paged via Gotify, and a cron job triggers an automated scrub + email digest. I've never lost data in 3 years of operation—though I came close once when two drives failed within 48 hours. Lesson learned: don't buy drives from the same batch."
